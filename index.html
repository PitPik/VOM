This demo initially shows a button with a text that represents the number of clicks being fired on itself which is 0 in the very beginning.

Initial buttons can be clicked 10 times before they would disappear again and can be restored by reloading the page (if there was no other button added like shown in the next step).

More buttons can be added by adding items to the model in the browser's console. 

demo.appendChild({
    clicks: 0,
    maxClicks: 12 // before it disappears
});

The state of an existing button can be manipulated in the browser's console.

demo.model[0].clicks = 8;
although, if the value is smaller or as big as the current value, an error will be shown in the console of the browser.

The states of the buttons are stored in the local storage and are restored on a new page request (refresh page).

With this demo, the comments inside demo.js and the documentation it should be easily possible to understand VOM and how it could be used for MVC patterns.
